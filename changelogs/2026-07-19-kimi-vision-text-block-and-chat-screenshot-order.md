# Kimi 识图文字块兼容 + 聊天截图顺序提取

**日期**：2026-07-19  
**涉及 commit**：`bdbcbb7`

## 改了什么
- 识图请求的 user content 从“只有 image_url”改成“text + image_url”，兼容 Kimi / 部分严格 OpenAI 兼容识图接口。
- 识图提示词增加聊天截图规则：按画面从上到下的时间顺序提取，不按左右两边分别归堆。

## 动了哪些文件
- `hooks/useChatAI.ts` —— 调整识图 prompt 和 visionMessages 请求格式。

## 踩坑 / 需要知道的（重要）
- Kimi 类接口可能不接受只有图片块的 user 消息，补一个文字块后更符合多模态常见格式。
- 聊天截图 OCR 容易把左侧气泡和右侧气泡分开读，导致主聊天模型理解成顺序错乱；提示词里必须明确“从上到下”。

## 备注
- 未改识图模型配置 UI。
- 未改主聊天图片注入逻辑，只改识图 API 调用前的描述生成。
